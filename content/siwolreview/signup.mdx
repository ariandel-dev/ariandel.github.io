### API 설계

가입시 이메일, 문자 관련 수신 동의하는 부분이 있는데
이런부분도 db에 저장을 해야한다.

동의하는 정책들이 고정되어있어서 현재 서비스에서는
user 엔티티에 동의내용을 포함하고 있지만 다른 서비스에서
어떻게 사용하는지 기록하기 위해 남겨둔다.

opgg같은 경우 정책이 자주 바뀌고 해서 버전 관리까지 해주는 것 같다.  
revu는 sms_subscribe = '1' 이런식으로 비교적 간단하게 관리하고 있다.

다만 두 서비스 모두 요청 url이 api.account.revu.net 이런식으로 되어있는데 아마 마이크로 서비스를 채택한 것 같다.
