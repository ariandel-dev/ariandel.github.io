## ECMAScript 버전차이 요약

### 클래스 선언

es5에서는 클래스 선언이 없었고 프로토타입 기반으로 구현되었다.

### 변수 선언

es5에서는 var로 선언되었고 es6에서는 let, const로 선언되었다.
스코프또한 블록스코프가 아니라 함수스코프였다.

### 화살표 함수

es5에서는 function 키워드로 선언되었고 es6에서는 화살표 함수로 선언되었다.
this 바인딩이 다르다. es6는 화살표 함수 자체의 this가 없으면 상위 스코프의 this를 참조한다.

### 템플릿 리터럴

es5에서는 문자열 연결을 위해 + 를 사용했지만 es6에서는 템플릿 리터럴을 사용할 수 있다.

### 디스트럭처링

es5에서는 객체 프로퍼티를 추출할 때 배열은 순서대로 추출하고 객체는 키를 지정해서 추출했다.
es6에서는 디스트럭처링을 사용할 수 있다.

### 모듈

es5에서는 모듈 시스템이 없었고 es6에서는 import, export를 사용할 수 있다.

### promise

es5에서는 콜백 지옥을 피하기 위해 라이브러리를 사용했지만 es6에서는 promise를 사용할 수 있다.
이후 es8에서는 async, await를 사용할 수 있다.

## 모듈 차이정리

| 특징 | CommonJS (CJS) | ES Module (ESM) |
| ---- | -------------- | --------------- |
| 지원된 시기 | ES5(Node.js) | ES6(ES2015) |
| 문법 | require(), module.exports | import/export |
| 실행 방식 | 동기적 (즉시 실행) | 비동기적 (지연 로딩 가능) |
| 환경 | Node.js에서 기본 사용 | 브라우저, Node.js에서 사용 가능 |
| Tree Shaking | 지원 안 함 | 지원 (최적화 가능) |
